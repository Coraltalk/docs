On a [custom domain](/coraltalk-apis/custom-domains), students cannot use email-code or OAuth sign-in directly. This endpoint enrolls the student (if needed) and returns a short-lived Clerk sign-in token for the exam UI.

## Platform API vs this route

|              | [Platform API](/coraltalk-apis/overview)  | Student sign-in token                              |
| ------------ | ----------------------------------------- | -------------------------------------------------- |
| **Base URL** | `https://app.coraltalk.com/api/v1/{env}/` | Same Coraltalk app, **exam hostname**              |
| **Auth**     | `Authorization: Bearer ct_live_...`       | None (custom domain only in production; see below) |
| **Caller**   | Your server / integration                 | Exam page in the **student’s browser**             |
| **Purpose**  | Manage courses, classes, assessments      | Sign in a student taking an exam                   |

This is **not** a separate API product on `exams.yourschool.edu`. Custom domains point at the same deployment; only the **host** changes. Exam code should call a **same-origin** path:

```
POST /api/student/sign-in-token
```

That resolves to your custom domain in production (for example `https://exams.yourschool.edu/api/student/sign-in-token`) or to `https://app.coraltalk.com/...` when the student opens the exam on the platform host.

<Warning>
  In production, `POST /api/student/sign-in-token` on **`app.coraltalk.com`
  returns `403`**. It is only enabled for verified custom domains (and
  `localhost` in development). Do not call it from `app.coraltalk.com` in a live
  custom-domain exam flow.
</Warning>

Platform API keys must not be embedded in exam pages. Server integrations should keep using `/api/v1/...` on `app.coraltalk.com` to create classes and assessments; student sign-in stays on the exam host.

## Endpoints

| Method | Path                         | Description                                |
| ------ | ---------------------------- | ------------------------------------------ |
| `POST` | `/api/student/sign-in-token` | Provision student and return sign-in token |

## Create sign-in token

| Field           | Required | Description                                         |
| --------------- | -------- | --------------------------------------------------- |
| `email`         | Yes      | Student email                                       |
| `firstName`     | Yes      | Given name                                          |
| `lastName`      | Yes      | Family name                                         |
| `orgInternalId` | Yes      | Your student identifier (roll number, SIS id, etc.) |
| `classId`       | Yes      | Class ID                                            |
| `courseId`      | No       | Course ID                                           |

**Custom domain (production exam flow):**

```bash
curl -X POST "https://exams.yourschool.edu/api/student/sign-in-token" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "student@school.edu",
    "firstName": "Ada",
    "lastName": "Lovelace",
    "orgInternalId": "roll-12345",
    "classId": "CLASS_ID",
    "courseId": "COURSE_ID"
  }'
```

**Platform host (local development only):**

```bash
curl -X POST "http://localhost:3000/api/student/sign-in-token" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "student@school.edu",
    "firstName": "Ada",
    "lastName": "Lovelace",
    "orgInternalId": "roll-12345",
    "classId": "CLASS_ID"
  }'
```

### Response

```json
{
  "token": "signin_xxxxxxxx",
  "userId": "user_xxxxxxxx",
  "studentId": "674a1b2c3d4e5f6789012345",
  "created": true
}
```

| Field       | Description                                                             |
| ----------- | ----------------------------------------------------------------------- |
| `token`     | Pass to Clerk sign-in with `strategy: "ticket"` (expires in 10 minutes) |
| `userId`    | Clerk user ID                                                           |
| `studentId` | Coraltalk student enrollment ID                                         |
| `created`   | `true` if the student was newly enrolled in the class                   |

Use `classId` from [Classes](/coraltalk-apis/classes) or from the exam’s `taAgent` field when building your exam flow.
