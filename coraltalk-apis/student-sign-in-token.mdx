# Student sign-in token

Use this endpoint from your **server** (with a Platform API key) when students take exams on a [custom domain](/coraltalk-apis/custom-domains). Clerk satellite mode blocks normal email-code sign-in on the exam hostname; your backend calls this API, then passes the returned `token` to the exam page for Clerk `strategy: "ticket"`.

<Warning>
  Never embed API keys in the browser. Call this route from your backend only.
</Warning>

## Endpoints

| Method | Path | Description |
| ------ | ---- | ----------- |
| `POST` | `/students/sign-in-token` | Provision student and return sign-in token |

## Create sign-in token

| Field | Required | Description |
| ----- | -------- | ----------- |
| `email` | Yes | Student email |
| `firstName` | Yes | Given name |
| `lastName` | Yes | Family name |
| `orgInternalId` | Yes | Your student identifier (roll number, SIS id, etc.) |
| `classId` | Yes | Class ID |
| `courseId` | No | Course ID |

```bash
curl -X POST "https://app.coraltalk.com/api/v1/prod/students/sign-in-token" \
  -H "Authorization: Bearer ct_live_YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "student@school.edu",
    "firstName": "Ada",
    "lastName": "Lovelace",
    "orgInternalId": "roll-12345",
    "classId": "CLASS_ID",
    "courseId": "COURSE_ID"
  }'
```

### Response

```json
{
  "data": {
    "token": "signin_xxxxxxxx",
    "userId": "user_xxxxxxxx",
    "studentId": "674a1b2c3d4e5f6789012345",
    "created": true
  },
  "meta": {
    "requestId": "req_abc123"
  }
}
```

| Field | Description |
| ----- | ----------- |
| `token` | Pass to Clerk sign-in with `strategy: "ticket"` on the custom domain (expires in 10 minutes) |
| `userId` | Clerk user ID |
| `studentId` | Coraltalk student enrollment ID |
| `created` | `true` if the student was newly enrolled in the class |

The class must belong to your organization and match the API key environment (`dev` or `prod`). Use `classId` from [Classes](/coraltalk-apis/classes).
