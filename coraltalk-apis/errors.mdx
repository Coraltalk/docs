# Errors

The Platform API uses standard HTTP status codes and a consistent JSON error body.

## Error format

```json
{
  "error": {
    "code": "validation_error",
    "message": "name is required",
    "field": "name"
  },
  "meta": {
    "requestId": "req_abc123"
  }
}
```

## HTTP status codes

| Status | Meaning |
| ------ | ------- |
| `400` | Bad request — missing field or invalid payload |
| `401` | Unauthorized — invalid, revoked, or env-mismatched API key |
| `403` | Forbidden — resource belongs to another org, or your organization isn't on a volume-based pricing plan |
| `404` | Not found |
| `429` | Rate limit exceeded |
| `500` | Server error — retry with backoff |

## Error codes

The `error.code` field is stable — branch on it instead of parsing `message`.

| Code | Status | Meaning |
| ---- | ------ | ------- |
| `validation_error` | `400` | A field is missing or invalid. Check `error.field`. |
| `unauthorized` | `401` | API key is missing, revoked, or doesn't match the environment. |
| `forbidden` | `403` | The resource belongs to another organization. |
| `volume_pricing_required` | `403` | Your organization isn't on a volume-based pricing plan. Contact [support@coraltalk.com](mailto:support@coraltalk.com) to enable it. |
| `not_found` | `404` | The resource doesn't exist or isn't visible to this key. |
| `rate_limited` | `429` | Too many requests. Back off and retry. |

## Support

When reporting issues, include `meta.requestId`, HTTP method, path, and timestamp. Email [support@coraltalk.com](mailto:support@coraltalk.com).
